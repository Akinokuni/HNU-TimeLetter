# 交互设计与页面逻辑 (Interaction Design)

本文档定义了“与她的海大时光笺”的核心交互逻辑与用户体验设计。

## 1. 全局体验 (Global Experience)

### 1.1 开场仪式 (Intro Page)
用户进入网站时的第一印象，旨在建立沉浸感并传达项目理念。当前版本调整为单屏启封结构，不再包含首屏后的“关于项目”与“关于我们”滚动页面。

#### 启封主屏 (The Envelope)
- **视觉**: 
  - **网格**: 首屏基于五列竖向网格排版，丝带贴合第一列右边界，标题居于第二列，信封居于右侧三列的中心。
  - **标题**: 左侧采用竖排标题“与她的海大时光笺”，与整高红色丝带并置，形成首屏识别锚点。
  - **核心**: 右侧展示信封主体，火漆印使用实物质感素材，信封采用固定尺寸主视觉。
- **交互**: 
  - **点击信封**: 
    - **动画**: 火漆碎裂 -> 信纸抽出 -> 整体淡出 (Framer Motion)。
    - **状态**: 动画结束后，全局状态 `isEnvelopeOpened` 设为 `true`，展示主界面（地图/集邮册）。
- **入场动效**:
  - **丝带**: 首次进入页面时基于 `clip-path` 执行自上而下的垂直裁剪显影，不改变位置与尺寸，采用双层 `requestAnimationFrame` 触发，时长 `1400ms`，缓动使用 `expo.out / power4.out` 风格减速曲线。
  - **信封**: 与丝带同时启动，遵循“离屏就绪 -> 主轴下坠与次级飘动 -> 阻尼落定”的三阶段入场，整体时长 `2000ms`。
  - **标题 / Logo**: 丝带完成后再执行淡入，时长 `800ms`，采用缓入缓出五次缓动。
- **滚动策略**:
  - 首屏不承担额外信息介绍职责。
  - 进入站点后的背景说明、团队信息等内容，后续应以独立模块或独立页面承载，而不是继续挂在启封页下方。

### 1.2 响应式分流 (Responsive Strategy)
采用 **双端差异化体验** 策略，而非简单的 CSS 响应式缩放。
- **PC 端 (>768px)**: 进入 **[时光画卷 (Desktop View)](#2-pc-端-时光画卷)**。
- **移动端 (≤768px)**: 进入 **[掌心集邮册 (Mobile View)](#3-移动端-掌心集邮册)**。
- **实现**: 在 `app/page.tsx` 中根据屏幕宽度条件渲染不同组件树。

## 2. PC 端: 时光画卷 (Desktop View)

> **核心隐喻**: “横向展开的时光画轴” —— 沉浸、动态、叙事。

### 2.1 交互式地图与卷轴 (Interactive Map & Scroll)
- **布局策略**: 
  - **初始状态 (Idle)**: 地图全屏显示，用户可以在地图上寻找地点。
  - **卷起动作 (Rolling)**: 点击地点 Pin 后，地图从左侧向右侧水平收纳，最终在右侧边缘形成一个宽 56px 的 **[卷轴条 (Scroll Strip)](#23-卷轴条视觉设计)**。
  - **收纳逻辑**: 地图容器使用 `overflow-hidden` 裁剪，内层地图保持 `100vw` 宽度并右锚定，确保收纳过程中 Pin 点的相对坐标依然准确。
- **层级关系 (Layering)**:
  - 为了保证交互体验，**故事面板 (z-30)** 位于 **地图/卷轴容器 (z-10)** 之上。
  - 这确保了在故事面板中拖拽卡片时，卡片可以自由覆盖在卷轴条上，不会产生视觉遮挡或被裁剪。

### 2.2 故事看板 (Story Stage)
- **出现时序**: 
  - **顺序感控制**: 故事看板不会在地图开始卷起时立即出现，而是等待地图完全收纳成卷轴（即 `rolled` 状态）后才开始进场。
  - **进场动画**: 采用从下方弹出的弹簧动画（Spring），无内部延迟，确保地图卷完后立即无缝衔接。
- **核心交互**:
  - **堆叠卡片 (Card Stack)**: 位于看板中心，支持向左/向右拖拽飞出，循环展示。
  - **底层卷轴**: 卷轴条始终位于右侧背景，作为返回地图的常驻入口。

### 2.3 卷轴条交互 (Scroll Strip Interaction)
- **还原逻辑**: 点击右侧卷轴条触发 `unrolling` 状态。
- **顺序感**: 
  1. 故事看板首先执行退场动画（向上飞出并淡出）。
  2. 看板完全退出后，地图容器开始向左展开恢复至 `100%` 宽度。
  3. 状态重置为 `idle`，激活地点清空。

## 3. 移动端: 掌心集邮册 (Mobile View)

> **核心隐喻**: “手中的集邮册” —— 高效、私密、易读。

### 3.1 故事瀑布流 (Story Feed)
- **策略**: 放弃手机端难用的地图交互，改为垂直瀑布流。
- **卡片**: 展示高清大图缩略 + 地点/角色信息。
- **FAB (悬浮按钮)**: 右下角常驻“地图”图标按钮，点击弹出一张 **纯静态海大地图** 图片，仅供查看地理位置，不可交互。

### 3.2 沉浸阅读 (Detail Modal)
- **转场**: 点击列表卡片 -> **共享元素转场 (Shared Element Transition)** -> 图片放大至全屏。
- **布局**: 上图下文。
  - **上部**: 高清大图 (占据约 50% 高度)，覆盖左右切换箭头。
  - **中部**: 角色头像与信息区。
  - **下部**: 滚动阅读故事文本。
- **手势**:
  - **点击箭头**: 切换同一地点的不同故事。
  - **左右滑动**: 切换不同地点。
  - **下拉 (Pull-to-close)**: 关闭详情页返回列表。

## 4. 下滚页面群 (Scroll-down Pages)

> **核心隐喻**：信封拆开后，视口向下延伸的"展览走廊"——由地图/集邮册继续向下滚动即可抵达。

### 4.0 全局滚动系统

#### 虚拟滚动动量 (Virtual Scroll Momentum)
- **基础位移**：遵循线性插值（Linear Interpolation），滚动多少、移动多少。
- **视觉优化**：外层套用虚拟滚动动量系统，带有微小的物理阻尼（Friction）和惯性（Inertia），使视觉上极为丝滑。
- **实现**：使用 `lerpFactor` 线性插值 + `requestAnimationFrame` 驱动，目标滚动位置与实际渲染位置之间有阻尼衰减。
- **技术选型**：纯原生 JS + React Hook，不引入额外库。

#### 红色引导线 (Red Guide Line)
- **视觉**：横跨三个页面的折线引导线，宽度 `100px`，颜色 `#c23643`，端点圆角（`stroke-linecap: round`, `stroke-linejoin: round`）。
- **层级**：位于最底层，仅在背景之上（`z-index: 1`）。
- **路径定义**（坐标为占比，页面左上角为 `0%, 0%`）：
  - 关于企划页：P1(17.396%, 0%) → P2(17.396%, 0.5%) → P3(19.15%, 7.50%) → P4(107.5%, 40.43%)
  - 鸣谢页：P5(-2.19%, 65.40%) → P6(32%, 126.27%)
- **动效逻辑**：非可逆单向线性插值（Non-reversible Linear Interpolation）
  - 使用 `Math.max(cachedMaxScroll, currentScroll)` 作为绝对进度标量
  - **向下滚动**：SVG 路径描边按比例实时正向曝光渲染
  - **向上回滚**：路径端点严格冻结在历史最远端，无反向收缩
  - **再次越过阈值**：插值计算无缝重连，继续正向曝光
- **实现**：SVG `<path>` + `stroke-dasharray` / `stroke-dashoffset`，通过滚动进度计算 offset。

## 5. 视觉风格 (Visual Style)

### 5.1 关键词
- **Galgame**: 精致人物立绘，情感化叙事。
- **Skeuomorphism (拟物)**: 信封、火漆印、邮票齿孔、纸张纹理。
- **Glassmorphism (玻璃拟态)**: 模态框背景采用高斯模糊 (Backdrop Blur)。

### 5.2 调色板 (Palette)
> 详见 `docs/design/视觉规范.md` 的 §2.1 色彩体系。以下为快速参考：
- **Primary**: `#c23643` (暗红/情书红)
- **Background**: `#ece9e4` (暖灰纸面)
- **Surface**: `#f6f1eb` (纸面卡片)
- **Ink Strong**: `#342628` (标题深墨色)
- **Ink**: `#5a4748` (正文默认色)
