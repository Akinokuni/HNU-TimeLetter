# 全站视口画框设计文档 (Site Viewport Frame)

> **状态**: 草案 / 文档先行定义
> **适用范围**: 全站路由、全局遮罩、模态层、加载态、导航层
> **核心规格**: 5px 白色 `#fffdfd` 边框、外轮廓直角贴合视口、内轮廓 16px 圆角裁切内容区

## 1. 方案定位

“全站视口画框”是站点级视觉骨架。页面内容、全局导航、模态层、过渡遮罩与加载态统一处于同一块圆角视口中，形成“纸面被安放在展示框内”的观感。

该方案服务于以下目标：

- **全站统一**: 首页 `/`、地图主体验 `/map`、创作公示板 `/creation`、后台 `/admin`、后台登录 `/admin/login`、404、加载态与全局弹层共享同一套画框规则。
- **内容内缩**: 画框不覆盖现有页面内容。页面主内容在布局层内缩，边框占据独立视觉空间。
- **固定层一致**: 所有 `fixed` / `inset-0` 型全屏层遵守相同的内缩边界与圆角裁切，不允许出现遮罩、弹层或滚动条破框。
- **沉浸感稳定**: 站点保留现有纸本叙事气质，同时通过统一边界强化“展陈式浏览”观感。

## 2. 视觉规格

### 2.1 画框本体

- **边框厚度**: `5px`
- **边框颜色**: `#fffdfd`
- **外轮廓圆角**: `0`（直角，贴合视口四边）
- **内轮廓圆角**: `16px`（画框内缘，裁切内容区）
- **内容区内缩量**: 与边框厚度一致，为 `5px`
- **画框形态**: 画框固定贴合视口四边，外四角为直角矩形；页面内容位于画框内部，由画框内缘的 16px 圆角反向裁切。

### 2.2 视觉关系

- 画框本身承担“站点最外层白色描边”的角色，外侧段落为直角矩形，取消视口四角的裸露区。
- 画框内部继续承载现有暖灰纸面背景与各页面自身的视觉内容。
- 页面内容区在视觉上比浏览器视口略小一圈，边框厚度即为站点与屏幕边缘之间的缓冲带。
- 画框内缘 16px 圆角需同时约束静态页面、滚动内容与全屏遮罩，形成统一的内轮廓。

### 2.3 设计 Token

全站画框采用以下 token，作为实现单一事实来源：

```css
--site-frame-border-width: 5px;
--site-frame-radius: 16px;
--site-frame-color: #fffdfd;
```

说明：

- `--site-frame-border-width` 表示内容内缩与边框厚度。
- `--site-frame-radius` 表示画框内缘的圆角半径（外轮廓始终为直角，不单独设 token）。
- `--site-frame-color` 表示全站唯一画框颜色，同时作为 html/body 背景，让视口直角四角与白边无缝衔接。

## 3. 适用范围

### 3.1 路由范围

| 路由 / 页面 | 画框要求 |
|-------------|----------|
| `/` | 首屏、下滚页面群与过渡层统一处于画框内 |
| `/map` | 地图主体验页面整体处于画框内，桌面端地图与移动端掌心集邮册共享统一外轮廓 |
| `/creation` | 创作公示板整体处于画框内，草稿线背景与便签瀑布流不得越过圆角 |
| `/admin/login` | 登录页整体处于画框内 |
| `/admin/*` | 后台侧栏、内容区与弹层统一处于画框内 |
| `loading.tsx` / `not-found.tsx` | 加载态与异常页遵守相同画框规格 |

### 3.2 全局层级范围

以下元素统一服从画框边界：

- **全局导航**: `GlobalNav` 的固定定位锚点相对于画框内部定位，不贴浏览器物理边缘。
- **过渡遮罩**: `TransitionOverlay` 与首页开信过渡层采用画框内全屏，不覆盖白色边框。
- **模态与抽屉**: Radix Dialog、移动端详情层、静态地图模态、搜索层等全屏遮罩遵守画框圆角。
- **自定义滚动条**: 滚动条轨道和滑块位于画框内部右侧，不压住边框。
- **全屏背景装饰**: 网格线、噪点、径向光、固定背景等若使用 `fixed` 定位，也服从画框内缩边界。

## 4. 实现策略

## 4.1 根布局职责

`src/app/layout.tsx` 负责承载站点级画框容器。

布局职责如下：

- `html` / `body` 提供全站外层背景色 `#fffdfd`。
- 站点主容器作为唯一的“画框视口容器”。
- `children`、`GlobalNav`、`TransitionOverlay` 统一挂在画框容器内，而不是散落在 `body` 根层。
- 任何需要“视口全屏”的层，都以画框容器为参照，而不是以浏览器物理边缘为参照。

### 4.2 全局样式职责

`src/app/globals.css` 负责定义全站画框 token 与公共类名。

推荐的公共类职责如下：

- **`site-frame-shell`**
  - 用于站点最外层容器。
  - 承担 `min-height`、`border-radius`、`overflow` 与背景裁切。
- **`site-frame-border`**
  - 画框描边层，位于视口最顶层，z-index 高于一切全屏遮罩。
  - 实现要素：容器 `position: fixed; inset: 0; overflow: hidden;` 提供视口直角裁切；
    内嵌 `::before` 以 `inset: var(--site-frame-border-width)` 与
    `border-radius: var(--site-frame-radius)` 作为内空洞几何；
    超大 spread 的 `box-shadow` 向外填满画框色，由容器 `overflow: hidden`
    裁剪至视口矩形。最终呈现：外角直角、内角 16px 圆角。
  - 禁用 `border` 属性的原因：Tailwind preflight 对 `*` 设置 `border-width: 0`，
    不同版本下与自定义类的级联顺序会导致设计值被覆盖，出现实际观察到
    1px 等不符规格的宽度。本方案仅使用 `box-shadow` 与 `overflow`，彻底绕开该冲突。

- **`site-frame-fixed`**
  - 用于所有原本写作 `fixed inset-0` 的层。
  - 统一改为以 `inset: var(--site-frame-border-width)` 参与布局。
  - 统一附带 `border-radius: var(--site-frame-radius)` 与 `overflow: hidden` 的圆角裁切能力。
- **`site-frame-scrollbar`**
  - 用于自定义滚动条轨道。
  - 轨道高度采用 `calc(100vh - var(--site-frame-border-width) * 2)`。
  - 轨道右侧位置与画框内边一致。

### 4.3 实现原则

- **原则一：不使用 `transform: scale()` 缩放整站**
  - 该项目包含 `Framer Motion`、Lenis、自定义滚动条与多个固定定位层。
  - 整站缩放会引入定位偏差、文字渲染模糊、交互命中偏移与滚动映射问题。
  - 画框采用“布局内缩 + 固定层统一 inset”的方式实现。

- **原则二：所有全屏层统一走共享类名**
  - 项目中已有多个 `fixed inset-0` 结构。
  - 单点手写尺寸会产生不一致与后续遗漏。
  - 画框相关的全屏层统一通过共享类名实现。

- **原则三：圆角裁切属于站点级约束**
  - 页面背景、图片、遮罩、粒子、装饰线都在圆角视口中展示。
  - 禁止让背景装饰绕开画框圆角直接贴到视口外缘。

### 5.2 高风险类型

以下结构是画框方案中的高风险项：

- **`fixed inset-0`**: 默认会贴满物理视口，必须收敛到共享类。
- **`h-screen` / `min-h-screen`**: 依赖视口高度的首屏模块需要确认在画框内是否仍满足视觉比例。
- **Portal 类模态**: 若内容通过 Portal 直接挂到 `body`，需要显式继承画框尺寸与圆角规则。
- **右侧固定锚点**: 导航栏、滚动条、浮动按钮等需避让 5px 画框宽度。

## 6. 页面级要求

### 6.1 首页 `/`

- 开场信封与下滚页面群共享同一画框轮廓。
- 开信过渡期间的遮罩层在画框内全屏铺开。
- 页脚揭露动效的固定层以画框内部为可视边界。
- 首页自定义滚动条在画框内部右侧运行。

### 6.2 地图主体验 `/map`

- 桌面端 `InteractiveMap` 与移动端 `MobileExperience` 共享统一的画框外轮廓。
- 地图底图、故事卡堆、移动端全屏体验层与相关模态遵守画框圆角与内缩边界。
- 从首页开信转入 `/map` 时，跨路由过渡遮罩保持画框边框完整可见。

### 6.3 创作公示板 `/creation`

- 草稿线背景与瀑布流容器在画框圆角内展示。
- 固定背景网格线不穿出圆角。
- 页面首屏与滚动区域在视觉上与首页保持统一外轮廓。

### 6.4 后台 `/admin`

- 登录页、侧边栏布局、内容区、弹层与反馈态统一纳入画框。
- 后台允许保留其信息密度与布局逻辑，但不脱离全站统一外轮廓。
- 后台固定侧栏的高度与定位参考画框内部，而非物理视口。

## 7. 非目标

以下内容不属于本方案目标：

- 浏览器地址栏、系统状态栏、设备安全区本身的外观控制。
- 针对单一路由设立独立画框规格。
- 通过缩放整个 DOM 树实现视觉内缩。
- 将画框设计拆分为首页专属效果。

## 8. 验收标准

### 8.1 视觉验收

- 站点四边始终可见 `5px` 白色 `#fffdfd` 边框。
- 站点视口四角为直角（贴合浏览器物理边框），不出现杂色或页面背景色裸露。
- 站点所有页面统一采用 `16px` 画框内缘圆角裁切内容区。
- 页面切换、过渡、加载、模态打开时，边框不被遮挡、不被染色、不被黑色遮罩覆盖。
- 自定义滚动条、导航栏、固定背景层与全屏模态都位于画框内部。

### 8.2 结构验收

- 画框相关尺寸由全局 token 统一控制，不允许在组件内分散硬编码。
- 全屏遮罩、Portal 模态与固定背景层具备统一的共享类或共享样式来源。
- `layout.tsx` 作为站点级画框容器的唯一挂载点。

### 8.3 交互验收

- Lenis 平滑滚动、首页转场、自定义滚动条与现有固定定位动画保持正常工作。
- 鼠标命中区域、滚动行为与动画坐标不因画框引入而发生偏移。
- 移动端模态与桌面端全屏搜索层在打开时不出现破框、露底或溢出。

## 9. 落地顺序

1. 在 `docs/design/` 固化方案文档，并挂入文档索引。
2. 在 `src/app/globals.css` 增加画框 token 与共享类。
3. 在 `src/app/layout.tsx` 建立站点级画框容器。
4. 将所有 `fixed inset-0` 型层迁移到共享画框类。
5. 调整导航栏、自定义滚动条与各类 Portal 遮罩的定位基准。
6. 逐页验证首页、创作公示板、后台、404、加载态与模态层表现。
