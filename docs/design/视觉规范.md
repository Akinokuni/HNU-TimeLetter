# 视觉设计规范 (Visual System)

> **目标**: 确立全局视觉基调，规范组件尺寸与状态，输出可开发的切图资源。

## 1. 风格关键词

- **Galgame**: 精致的人物立绘，强调情感氛围。
- **Skeuomorphism (拟物)**: 信封、火漆印、邮票齿孔、纸张纹理。
- **Glassmorphism (玻璃拟态)**: 模态框背景采用高斯模糊 (Backdrop Blur)，增加现代感。

## 2. 全局设计变量 (Design Tokens)

### 2.1 色彩体系 (Color Palette)

- **Primary (主色)**: `#0ea5e9` (海大蓝/Ocean Blue) - 用于强调、按钮、Link。
- **Background (背景)**: `#fdfbf7` (米白/信纸色) - 用于全局背景，营造温暖怀旧感。
- **Text (正文)**: `#292524` (Stone-800) - 偏暖深灰，阅读舒适。
- **Muted (辅助)**: `#78716c` (Stone-500) - 用于次要信息。
- **Border (边框)**: 待定

### 2.2 字体排印 (Typography)

- **Headings (标题)**: `Noto Serif SC` (宋体) - 营造文学感与回忆感。
- **Body (正文)**: `Noto Sans SC` (黑体) - 确保多终端可读性。
- **Handwriting (手写)**: 用于“背后的故事”或签名 (待定: WebFont 或 SVG)。

### 2.3 玻璃拟态与阴影

- **模态框背景**:
  - 透明度: `待定`
  - 模糊度 (Blur): `10px-20px`
  - 边框: 1px 白色半透明描边 (可选)
- **卡片阴影**: 需定义常规态与悬浮态 (Hover)。

## 3. 关键视觉资产清单 (Assets Checklist)

### 3.1 开场信封 (Envelope Animation)

为了实现“开信”动画，需要**分层素材**：
- [ ] **信封-背面 (Envelope Back)**: 主体容器。
- [ ] **信封-盖子 (Envelope Flap)**: 独立盖子图层（翻转动画用）。
- [ ] **信纸 (Letter Content)**: 稍微露出的信纸边缘。
- [ ] **火漆印 (Wax Seal)**: 海大校徽元素（独立图层，碎裂/淡出效果）。
- [ ] **完整闭合信封图**: 加载时静态展示。

### 3.2 海大地图 (The Map)

- **PC 端**:
  - 格式: SVG (最佳) 或 高清 PNG (宽 >= 2560px)。
  - 风格: 扁平化/手绘风/等轴测。
  - 注意: 四周预留背景色，防止裁切。
- **移动端**:
  - 格式: 竖版/方形静态图 (仅供查看)。

### 3.3 邮票与标记 (Stamps & Pins)

- **地图坐标点 (Pin)**:
  - Q版头像边框容器 (48x48 或 64x64)。
- **邮票样式 (Stamp Frame)**:
  - 齿孔边框素材 (中间镂空)。
  - 宽高比固定 (3:4 或 1:1)。

## 4. UI 组件规范

### 4.1 PC 端明信片模态框
- **布局**: 左图右文。
- **尺寸**: 固定宽高或 Max-width。
- **交互**: 左右切换箭头 (常规/悬浮/点击)。

### 4.2 移动端卡片流
- **封面图**: 统一比例 (4:3 或 16:9)。
- **详情页**: 上图下文。
- **FAB**: 右下角悬浮地图按钮。

## 5. 交付格式要求

1. **设计稿**: Figma (Dev Mode) 或 蓝湖。
2. **切图**:
   - 图标/Logo/线条 -> **SVG**。
   - 照片/插画 -> **WebP** / **PNG** (@2x/@3x)。
3. **命名**: `component_state.format` (e.g., `stamp_frame_active.png`).
